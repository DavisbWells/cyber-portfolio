Overview

A virtualized cybersecurity lab running on a dedicated host machine, designed to simulate a small enterprise network. The environment includes isolated network segments, a SIEM for log aggregation, and intentionally vulnerable machines for safe exploitation practice. All attack activity stays within the isolated lab network — no external traffic.

Current Experiments

Splunk SIEM Detection Rules

Writing and tuning SPL (Search Processing Language) queries to detect brute-force attempts, lateral movement via PsExec, and abnormal process creation events across Windows Server hosts.

pfSense IDS/IPS Tuning

Configuring Suricata rules on pfSense to detect C2 callback patterns, DNS tunneling, and port scan signatures. Reducing false positives while maintaining high detection fidelity.

Active Directory Attack & Defend

Simulating common AD attacks (Pass-the-Hash, Kerberoasting, DCSync) in the target VLAN, then building Splunk queries to detect each technique using Windows Event Log telemetry.

Packet Capture Analysis Pipeline

Automating PCAP collection from the network segment, parsing with Python/Scapy, and forwarding IOCs to a local threat intelligence database for correlation.
